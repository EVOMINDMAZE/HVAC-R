## Plan: Fix "Token Invalid" Error Handling

### Problem
When RPC calls like `use_invite_code` or `get_user_companies_v2` fail due to invalid/expired tokens, the error is not properly detected and handled, leaving users stuck.

### Root Cause
The current `AuthErrorHandler` only checks for specific phrases like "Invalid Refresh Token" but misses common Supabase auth errors such as:
- "jwt expired"
- "invalid JWT token"
- "token is expired"
- Authentication errors from RPC calls

### Solution

**1. Update `authErrorHandler.ts`**
- Expand token error patterns to include all common Supabase auth errors
- Export a helper function `isTokenError()` for use in components
- Improve global error handler reliability

**2. Update `JoinCompany.tsx`**
- Import and use `isTokenError()` in error handling
- When token error detected: call `signOut()` and redirect to `/signin`
- Show user-friendly message: "Your session has expired. Please sign in again."

**3. Update `useSupabaseAuth.tsx`**
- Add token error detection in `fetchCompanies()`
- When token error detected: trigger proper sign out and state cleanup
- Prevent infinite loading by handling errors gracefully

### Expected Outcome
Users with invalid/expired tokens will be:
1. Automatically signed out
2. Redirected to the sign-in page
3. Shown a clear error message about session expiration
